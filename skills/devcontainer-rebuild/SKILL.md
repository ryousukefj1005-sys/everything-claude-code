# Dev Container 重建技能

## 描述

重建当前项目的 Dev Container 环境。适用于依赖更新或容器出现问题时。

## 使用场景

- 更新 requirements.txt 后重新安装依赖
- 容器出现异常需要重建
- 切换分支后环境变化

## 执行步骤

1. 检查是否存在 .devcontainer 配置
2. 停止并删除当前容器
3. 重新构建容器镜像
4. 启动新容器
5. 验证环境是否正常

## 参数

无

## 示例

```
/devcontainer-rebuild
```

## 注意事项

- 重建过程需要 3-5 分钟
- 会清除容器内的所有临时文件
- 本地代码不会被删除（挂载在容器外）
